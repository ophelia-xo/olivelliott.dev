    1. **Strategy:** Hand-author paths that approximate the Geist Mono `oe` letterforms at the required scale. Geist Mono is monospace, so `o` and `e` each occupy a 12-unit wide cell within the 24×16 interior (12 + 12 = 24 wide; centered). Letterforms are amber `#fbbf24` on dark `#0a0a0a` background.

    2. **Body — verbatim:**
       ```xml
       <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32" width="32" height="32" role="img">
         <title>olivelliott.dev</title>
         <desc>Lowercase oe monogram in amber on dark — favicon for olivelliott.dev.</desc>
         <rect width="32" height="32" fill="#0a0a0a"/>
         <!-- Lowercase 'o' — circle approximation of Geist Mono o glyph -->
         <!-- Centered at (10, 18), outer radius 5, stroke-width via two concentric circles (donut). -->
         <path d="M 10 13 a 5 5 0 1 0 0.01 0 z M 10 15.5 a 2.5 2.5 0 1 1 -0.01 0 z" fill="#fbbf24" fill-rule="evenodd"/>
         <!-- Lowercase 'e' — Geist Mono e: circular bowl with horizontal crossbar. -->
         <!-- Centered at (22, 18), outer radius 5, crossbar at y=18. -->
         <path d="M 22 13 a 5 5 0 1 0 4.95 5.85 l -2.45 0 a 2.5 2.5 0 1 1 -0.05 -3.85 a 2.5 2.5 0 0 1 2.4 1.5 l -4.85 0 z" fill="#fbbf24"/>
       </svg>
       ```

    3. **Why path approximations, not pixel-perfect Geist glyphs:** Per UI-SPEC § Favicon, the glyph-as-path strategy freezes the exact mark shape so every browser renders identically. Hand-authored paths that LOOK like Geist Mono `oe` (round, monospace-grid, with the characteristic Geist `e` crossbar) satisfy the visual contract — at 16px in a browser tab strip, a 4-pixel-wide donut for `o` and a 4-pixel-wide bowl-plus-bar for `e` ARE the typographic signal. If Olive later wants pixel-exact Geist Mono glyph paths, regenerate `app/icon.svg` from a Figma export in a v1.x cleanup (UI-SPEC notes this is a Phase 7 follow-up if desired).

    4. **Color discipline (UI-SPEC § Color):** ONLY these two hex values appear in the SVG: `#0a0a0a` (background rect) and `#fbbf24` (letterform paths). NO accent stroke, NO accent dot, NO accent corner-mark, NO gradient, NO `<filter>`, NO `<animate>`. Lock-step with `styles/tokens.css` --color-bg + --color-accent.

    5. **A11y:** `<title>` + `<desc>` inside the SVG so screen readers exploring an embedded-inline favicon usage have a label. Browser-tab favicon rendering does not surface alt text — the title is for the embedded case.

    Final on-disk size target: < 2 KB (path-only SVG, no embedded raster).

    2. **OVERWRITE `app/favicon.ico` with a multi-size 16/32/48 raster bundle.** Sharp can output PNG buffers; assemble them into ICO byte structure manually:
       ```bash
       node -e "
         const sharp = require('sharp');
         const fs = require('fs');
         const svg = fs.readFileSync('app/icon.svg');
         async function buildIco() {
           const sizes = [16, 32, 48];
           const pngs = await Promise.all(
             sizes.map(s => sharp(svg).resize(s, s, { kernel: 'nearest' }).png({ compressionLevel: 9 }).toBuffer())
           );
           // ICONDIR header (6 bytes) + N×ICONDIRENTRY (16 bytes each) + concatenated PNG data
           const header = Buffer.alloc(6);
           header.writeUInt16LE(0, 0);     // reserved
           header.writeUInt16LE(1, 2);     // type 1 = icon
           header.writeUInt16LE(sizes.length, 4); // image count
           const dir = Buffer.alloc(16 * sizes.length);
           let offset = 6 + 16 * sizes.length;
           sizes.forEach((s, i) => {
             const o = i * 16;
             dir.writeUInt8(s === 256 ? 0 : s, o + 0);     // width
             dir.writeUInt8(s === 256 ? 0 : s, o + 1);     // height
             dir.writeUInt8(0, o + 2);                      // palette count
             dir.writeUInt8(0, o + 3);                      // reserved
             dir.writeUInt16LE(1, o + 4);                   // color planes
             dir.writeUInt16LE(32, o + 6);                  // bits per pixel
             dir.writeUInt32LE(pngs[i].length, o + 8);      // image size
             dir.writeUInt32LE(offset, o + 12);             // image offset
             offset += pngs[i].length;
           });
           const ico = Buffer.concat([header, dir, ...pngs]);
           fs.writeFileSync('app/favicon.ico', ico);
           console.log('favicon.ico OK — sizes:', sizes.join('/'), '— bytes:', ico.length);
         }
         buildIco().catch(e => { console.error(e); process.exit(1); });
       "
       ```
